# 团队分工

| 成员 | 负责模块与职责 |
| :--- | :--- |
| **黄嘉禾** | 负责基础任务智能体开发部署，筛选“安全报告自动生成”、“告警误报剔除”、“漏洞排查与闭环”等高频真实工业界场景，利用深信服智能体平台进行开发 |
| **黄家骏** | 负责系统核心后端架构的设计与开发，以及进阶任务的工具扩展，开发自定义数据接口，对接防火墙封禁、EDR隔离等第三方安全工具 API |

## 初版任务点

* **黄嘉禾｜基础任务智能体与安全场景负责人**

  * 负责梳理工业界高频安全场景，优先完成“安全报告自动生成”“告警误报剔除”“漏洞排查与闭环”三个基础任务。
  * 负责为每个场景编写需求说明，明确输入数据、输出结果、使用角色、触发方式、执行边界和验收指标。
  * 负责调研深信服智能体平台现有能力，包括模型调用、知识库、工作流、插件、工具调用、权限控制和发布方式。
  * 负责确定哪些能力直接使用深信服平台实现，哪些能力需要调用团队自建后端服务。
  * 负责设计三个基础智能体的系统提示词、任务提示词、输出格式和异常处理提示词。
  * 负责设计“安全报告自动生成”智能体的输入模板，包括告警摘要、资产信息、攻击时间线、处置记录和影响范围。
  * 负责设计安全报告的标准输出结构，包括事件概述、攻击链分析、影响评估、处置过程、证据列表、遗留风险和整改建议。
  * 负责设计“告警误报剔除”智能体的判断流程，包括告警上下文补全、资产重要性判断、历史行为比对和误报理由生成。
  * 负责定义告警分类结果，例如“确认攻击”“高度可疑”“低风险异常”“误报”“信息不足”。
  * 负责设计“漏洞排查与闭环”智能体的业务流程，包括漏洞发现、资产确认、风险评级、修复建议、修复验证和关闭工单。
  * 负责为基础场景设计工具调用清单，明确每个工具的用途、输入参数、输出格式和调用权限。
  * 负责配置深信服智能体平台中的基础工作流、知识库连接、工具插件和模型参数。
  * 负责开发基础场景的 Demo，并保证能够使用模拟数据完成端到端演示。
  * 负责整理常见安全术语、报告模板、误报案例和漏洞修复模板，交由刘家源纳入知识库。
  * 负责建立基础任务测试集，每个场景至少包含正常样例、边界样例、异常样例和恶意输入样例。
  * 负责定义基础任务的验收指标，包括任务完成率、格式正确率、事实一致性、工具调用成功率和人工认可率。
  * 负责对模型输出进行安全性检查，避免智能体直接执行高风险操作。
  * 负责设计高风险动作的人工确认节点，例如防火墙封禁、EDR隔离、账号禁用和漏洞关闭。
  * 负责定义基础智能体的业务降级策略，例如工具不可用时只生成处置建议，不执行实际动作。
  * 负责记录深信服平台开发过程中遇到的限制，为后续 LangGraph 和 Eino 实现提供需求依据。
  * 负责基础场景的演示脚本、操作手册和业务说明文档。
  * 负责基础场景最终验收，并确认生产版智能体是否满足真实安全运营人员的使用需求。
  * 与黄家骏共同确定基础智能体调用后端接口的参数格式。
  * 与岳晓峰共同确定基础场景中需要进入复杂规划流程的触发条件。
  * 与刘家源共同制定安全知识库内容范围和知识更新频率。
  * 与伍杜烽共同完成基础智能体的压力测试、稳定性测试和部署验证。
  * 主要交付物包括：场景需求文档、智能体提示词、深信服平台工作流、基础场景测试集、验收报告、演示脚本和用户操作手册。

* **黄家骏｜系统架构、后端接口、工具接入与 Go/Eino 生产化负责人**

  * 负责设计系统整体技术架构，明确前端、智能体编排层、模型服务层、RAG服务层、工具服务层、数据层和第三方安全设备之间的关系。
  * 负责绘制系统架构图、部署架构图、数据流图和外部系统交互图。
  * 负责确定后端服务拆分方案，例如智能体服务、任务服务、工具网关、知识检索服务、审计服务和配置服务。
  * 负责制定项目统一的 RESTful API 标准。
  * 负责规定 API 路径命名、HTTP方法、请求参数、响应结构、分页方式、错误码和版本管理方式。
  * 负责制定统一响应格式，例如 `code`、`message`、`data`、`request_id` 和 `timestamp`。
  * 负责制定接口字段命名规范，统一使用蛇形命名或驼峰命名。
  * 负责制定接口幂等机制，避免封禁、隔离和工单关闭等动作被重复执行。
  * 负责设计异步任务接口，包括任务创建、任务查询、任务取消、任务重试和任务结果获取。
  * 负责设计智能体流式输出接口，例如使用 SSE 或 WebSocket 返回执行状态、工具调用和最终结论。
  * 负责设计 OpenAPI/Swagger 文档，并保证所有后端接口能够自动生成接口说明。
  * 负责设计 API 鉴权机制，包括用户身份认证、服务间认证、API Key 管理和 Token 过期策略。
  * 负责设计基于角色的权限控制，区分普通分析员、安全管理员、系统管理员和审计人员。
  * 负责设计工具调用网关，所有高风险安全工具必须通过统一网关访问。
  * 负责定义统一的工具接口协议，包括工具名称、工具描述、输入模式、输出模式、超时时间、重试次数和风险等级。
  * 负责开发防火墙封禁接口适配器，实现IP封禁、域名封禁、封禁查询和解除封禁。
  * 负责开发 EDR 接口适配器，实现终端查询、进程查询、终端隔离和解除隔离。
  * 负责开发漏洞扫描平台接口适配器，实现漏洞查询、任务创建、扫描状态查询和扫描结果获取。
  * 负责开发工单系统接口适配器，实现工单创建、负责人分配、状态更新、评论追加和关闭工单。
  * 负责开发 SIEM、日志平台或告警中心的数据查询接口。
  * 负责第三方接口的认证信息加密存储，不允许在代码或配置文件中明文保存密钥。
  * 负责设计工具调用的超时、限流、熔断、重试和失败降级机制。
  * 负责设计危险动作的二次确认接口，未经人工审批不得调用生产安全设备。
  * 负责建立工具调用审计日志，记录调用人、智能体、参数摘要、执行时间、执行结果和审批信息。
  * 负责设计后端任务状态持久化方案，保存智能体当前状态、历史步骤、工具结果和重试次数。
  * 负责选择并配置关系数据库，用于保存用户、任务、配置、审批记录和审计信息。
  * 负责设计 Redis 的使用方式，包括任务锁、临时状态、缓存、限流和分布式协调。
  * 负责设计消息队列或任务队列，用于处理耗时安全任务和异步工具调用。
  * 负责设计模型服务的统一调用接口，使 LangGraph、深信服平台和 Eino 使用相同的模型访问协议。
  * 负责设计模型路由策略，根据任务复杂度、延迟要求和成本选择不同模型。
  * 负责设计调用链追踪机制，为每次请求生成唯一的 `trace_id` 和 `request_id`。
  * 负责实现后端日志结构化输出，便于日志平台检索和问题定位。
  * 负责定义敏感字段脱敏规范，包括账号、IP、主机名、Token和设备标识。
  * 负责审查所有外部输入，防止SQL注入、命令注入、路径遍历、SSRF和参数越权。
  * 负责 LangGraph 原型系统与 Go/Eino 生产系统之间的接口兼容性设计。
  * 负责制定 LangGraph 到 Eino 的迁移计划，明确哪些组件直接重写，哪些组件通过接口复用。
  * 负责将 LangGraph 中的节点映射为 Eino 中的组件、工具、链、图节点或工作流单元。
  * 负责将 LangGraph 中的状态对象转换为 Go 结构体，并定义字段校验、序列化和版本兼容策略。
  * 负责在 Go 中实现智能体运行时、工具注册、上下文传递、错误处理和并发控制。
  * 负责实现 Eino 生产版智能体的节点超时、节点重试、任务恢复和状态持久化。
  * 负责确保 Go/Eino 版本与 LangGraph 原型在相同测试集上获得等价结果。
  * 负责生产版后端服务的性能优化，包括连接池、协程控制、对象复用和缓存策略。
  * 负责维护后端 README、接口文档、架构决策记录和迁移说明。
  * 与岳晓峰共同确认状态图在后端中的存储和执行方式。
  * 与刘家源共同确定 RAG 服务接口、检索参数和引用返回格式。
  * 与伍杜烽共同完成接口联调、Go代码审查、容器化和生产部署。
  * 主要交付物包括：系统架构文档、OpenAPI文档、后端服务、工具网关、第三方安全工具适配器、权限系统、审计系统、Go/Eino生产版智能体和迁移报告。

* **岳晓峰｜复杂任务编排、ReAct自主规划与前端可视化负责人**

  * 负责分析复杂安全任务的执行流程，抽象智能体需要维护的状态、事件、节点和状态转移条件。
  * 负责设计 LangGraph 的全局状态结构，包括用户目标、当前计划、已完成步骤、待执行步骤、工具结果、风险等级、错误信息和最终结论。
  * 负责绘制智能体状态转移图，明确每个节点的进入条件、退出条件和异常分支。
  * 负责定义初始状态、规划状态、工具执行状态、观察状态、反思状态、重新规划状态、人工审批状态、结束状态和失败状态。
  * 负责确定哪些节点由大模型执行，哪些节点由确定性代码执行。
  * 负责设计复杂任务的 Planner 节点，将用户目标拆分为可执行的安全子任务。
  * 负责设计 Executor 节点，根据规划选择并调用合适的安全工具。
  * 负责设计 Observer 节点，对工具返回结果进行标准化和有效性检查。
  * 负责设计 Critic 或 Reflector 节点，判断当前结果是否足以完成任务。
  * 负责设计 Replanner 节点，在工具失败、信息不足或结果冲突时重新生成计划。
  * 负责定义智能体终止条件，防止无限循环和无效重复调用。
  * 负责定义最大规划轮数、最大工具调用次数和最大任务执行时间。
  * 负责设计工具调用前的风险判断节点，将操作划分为只读、低风险写入和高风险执行。
  * 负责设计 Human-in-the-loop 节点，在封禁、隔离、删除和关闭工单前请求人工审批。
  * 负责设计计划回滚逻辑，例如解除错误封禁、撤销隔离和恢复任务状态。
  * 负责设计跨域协同流程，例如从告警分析扩展到资产查询、威胁情报、漏洞扫描、终端隔离和工单闭环。
  * 负责设计多智能体之间的任务分派机制，例如告警分析智能体、漏洞分析智能体、处置智能体和报告生成智能体。
  * 负责定义多智能体之间的消息格式、上下文共享范围和结果汇总规则。
  * 负责防止多个智能体重复执行同一个高风险动作。
  * 负责设计长任务的断点恢复机制，使任务中断后可以从最近状态继续执行。
  * 负责设计推理过程的结构化记录，只保留任务计划、证据、工具结果和决策摘要。
  * 不在日志、数据库或前端中直接暴露模型原始思维链，只展示可审计的结构化决策依据。
  * 负责设计前端总体信息架构，包括任务创建页、任务执行页、告警分析页、知识检索页、审批页和审计页。
  * 负责任务执行过程的可视化，展示当前节点、已完成节点、待执行节点和失败节点。
  * 负责设计攻击链或处置链路可视化，展示告警、资产、漏洞、威胁情报和处置动作之间的关系。
  * 负责设计工具调用详情面板，显示工具名称、输入摘要、执行状态、耗时和输出摘要。
  * 负责设计人工审批界面，展示拟执行动作、风险说明、证据来源和回滚方案。
  * 负责设计安全报告的前端预览、编辑和导出功能。
  * 负责设计任务失败后的前端提示，区分模型失败、工具失败、权限不足和数据不足。
  * 负责设计 WebSocket 或 SSE 消息在前端的状态处理逻辑。
  * 负责定义前后端状态字段，避免前端自行猜测智能体状态。
  * 负责设计复杂任务测试案例，包括工具连续失败、检索结果冲突、错误计划和人工拒绝审批。
  * 负责验证智能体是否真正具备观察、反馈、反思和重新规划能力。
  * 负责输出 LangGraph 节点设计文档，供黄家骏迁移到 Eino。
  * 负责提供 LangGraph 节点和 Eino 节点之间的行为对照表。
  * 与黄嘉禾共同确定基础任务升级为复杂自主规划任务的条件。
  * 与黄家骏共同制定状态数据结构、异步执行接口和流式消息协议。
  * 与刘家源共同确定知识检索节点在状态图中的位置和触发条件。
  * 与伍杜烽共同完成复杂流程的自动化回归测试和故障注入测试。
  * 主要交付物包括：LangGraph状态图、状态模型、ReAct规划流程、多智能体协同机制、前端系统、执行过程可视化、复杂任务测试集和Eino迁移行为说明。

* **刘家源｜安全数据治理、领域知识增强、RAG与评测负责人**

  * 负责确定安全知识库的数据范围，包括漏洞库、攻击技术、威胁情报、安全设备文档、处置手册和历史安全事件。
  * 负责收集公开安全数据，例如CVE、CWE、CAPEC、ATT&CK、厂商安全公告和安全最佳实践。
  * 负责收集团队内部数据，例如历史告警、历史工单、处置记录、误报案例和安全报告。
  * 负责建立数据来源清单，记录来源、授权方式、更新时间、可信度和使用限制。
  * 负责对敏感数据进行脱敏，清除账号、Token、个人信息、内部域名和不应暴露的资产信息。
  * 负责设计安全数据清洗流程，包括去重、乱码清理、格式统一、无效内容过滤和异常样本处理。
  * 负责设计文档标准化格式，统一保存标题、正文、来源、时间、标签、权限和版本信息。

  * 负责选择文档切分策略，根据安全文档类型使用固定长度、标题层级或语义切分。

  * 负责选择和评估向量嵌入模型，重点考察中文安全语义、英文漏洞描述和中英文混合检索能力。

  * 负责设计向量库集合或索引划分方式，避免不同权限、不同来源的数据互相污染。
  * 负责实现文档导入、增量更新、删除、重建索引和版本回滚流程。
  * 负责实现关键词检索、向量检索和混合检索。
  * 负责实现元数据过滤，例如按照漏洞类型、产品、时间、严重等级和数据来源过滤。
